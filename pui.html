Bun Bun Bake Shop

Overview
We were tasked with prototyping, testing, and implementing a site for a fictitious bakery, Bun Bun Bake Shop in our Programming Usable Interfaces course. Using a combination of HTML, CSS, and JavaScript, I created a site in which customers could add and remove items from their shopping cart.

Project Details
Context: Programming Usable Interfaces, CMU
Timeline: September-October, 2020
GitHub Repository

Prompt
For our first major coding assignment, we set out to build a site from the ground-up, starting with prototypes and eventually working our way to a functioning site. In the final site, a user could select the flavor, glaze, quantity, and pickup option for cinnamon rolls and add them to the cart. Despite my computer science background, this was my first time working with with HTML, CSS, and JavaScript. How different could it be?

Reflection
In short: it is very different.
My biggest challenges were in implementing the JavaScript functionality. Originally, I had a somewhat hack-y way to count the number of items added into the cart. I was able to eventually find a better solution, but it took a lot of debugging along the way to solve it (console.log() is my friend ❤️).

Process

Paper Prototypes
The first step was to build paper prototypes of the home, product browsing, and product information pages. I tested them with various users as well as conducted competitive analysis to generate three iterations. I found that it’s best to keep it simple and straightforward in the layout and options given to users to minimize potential errors or confusion.

High-Fidelity Prototype
Once the paper prototypes had been tested and gone through several iterations, it was time to create a high-fidelity prototype of the Bun Bun Bake Shop website. My foremost considerations were toward balancing customization options and error prevention. As a result, the prototype uses a simplified structure that is complemented by visual components that call to mind the light-hearted and informal mood of a bakery. The key takeaway of my design is that the straightforward structure and user actions in combination with the aesthetic design can work hand-in-hand to create a usable and enjoyable prototype.

HTML & CSS
When I had originally set out to code the HTML and CSS of the site, I was unfamiliar with the conventions and style guides to follow when formatting. With the help of classmates and a lot of Google searching, I was able to utilize a grid to help my layout and position elements. Additionally, I learned when and how to use divs effectively, which helped clean up my code and format pictures correctly.
See my GitHub repository link in the Overview section for the final code!

Javascript
In order to implement the functionality of the site to customize cinnamon roll orders and add or remove them from the cart, I wrote Javascript functions using local storage and JSON objects. It involved a lot more Google searches and re-writing code when I realized there was a more efficient way to achieve the same end result. Ultimately, while it was challenging, it was incredibly satisfying when the functions worked the way they were supposed to!

Final Site
Overall, I'm happy with my final site, especially as my first foray into the world of HTML, CSS, and Javascript. I learned to use the resources at my disposal to resolve issues that I came across, and to debug as much as possible. I think the design elements in combination with the functionality of selecting items and adding them to the cart helped create a site that is user- friendly and reflected the small-business, handmade feel of the bakery.
